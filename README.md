# 还愿日记 · 四面佛祖 × 数字记忆

三宝佛光寺四面佛祖的「祈愿—还愿」内容系统。

给信众一个地方**许下心愿、等待应验、回来写故事**；给寺院一个**自动采集、可持续运营**的内容库。

---

## 这是什么

一套免费、开源、无需服务器的**还愿日记系统**。

**现象**：信众在还愿时最动人的故事（从许愿到应验的那段心路），往往一刹那就消散了。四面佛祖有那么多回应，为什么不被记住？

**解决**：信众用手机扫码进入「还愿日记」小程序，不需真名、自取化名。一路引导：
- 「许愿区」：选四面中的哪一面、写下心愿、获得凭证号
- 「我的专区」：回来还愿时，凭证号找回自己的心愿、写下故事
- 「故事卡」：一张可截图分享、可传播的「还愿日记」

寺院则得到：
- 一张 Google 表格，自动记录每一条许愿和还愿
- 一个「同意公开分享」的勾选框，精准筛选可用于内容运营的故事
- 随时可用这些**真实、有温度的故事**做成短视频、微博、故事集

## 快速开始（5 分钟）

### 先试用（不需连表格）
1. 下载 `huanyuan-riji.html`
2. 用浏览器打开
3. 按"许下心愿" → 填信息 → 看演示完整流程（数据暂存本机）

### 真正上线（15 分钟）
详见 `部署说明.md`：
1. 建一张 Google 表格
2. 贴后端代码 `Code.gs` 到 Google Apps Script
3. 部署成 Web 应用，复制网址
4. 网址粘进 HTML 里的 `APPS_SCRIPT_URL` 配置
5. 把 HTML 上传到寺院网站，或任何静态托管（Netlify、GitHub Pages、Vercel）
6. 在还愿区贴二维码，让信众扫码

完成。每一次许愿 / 还愿都自动到表格。

## 系统说明

### 三个文件

| 文件 | 用途 | 谁用 |
|------|------|------|
| `huanyuan-riji.html` | 前端页面（信众看到的界面） | 信众手机打开 |
| `Code.gs` | Google Apps Script 后端 | 部署在 Google Apps Script |
| `部署说明.md` | 一步步部署指南 | 寺院技术负责人 |

### 信息流

```
信众手机
  ↓ (许愿/还愿表单 JSON)
huanyuan-riji.html (前端)
  ↓ (POST)
Google Apps Script (Code.gs)
  ↓ (appendRow)
Google Sheets (你的表格)
  
信众换手机 / 凭证号找回
  ↓ (GET ?id=XXXXX)
Google Apps Script (JSONP 回源)
  ↓
huanyuan-riji.html (本地还原)
```

### 核心功能

#### 给信众

✅ **匿名友好**
- 不需真名，自取化名（内置 20 个诗意化名选项或自己编）
- 无需登录，无需留电话

✅ **四面选择**  
- 直观的四方向选择器，轻触选择「事业/姻缘/财富/健康」或寺院自定义的四面含义
- 下方实时显示该面所求

✅ **凭证号体系**
- 许愿后生成 6 位凭证号（如 `HYP9MY`），截图保存
- 同一部手机自动记住所有心愿
- 换了手机输入凭证号即可找回（前提是部署了后端）

✅ **故事写作**
- 还愿时用引导式提问：等了多久 → 怎么应验的 → 想说的一句话
- 缓解"我不知道怎么表达"的焦虑
- 自动生成一张可截图、可分享的「还愿日记」故事卡

✅ **隐私勾选**
- "我愿意让寺院分享我的化名故事"
- 只有勾选"是"的故事，寺院才能公开使用
- 即使公开也只显示化名，隐去真实身份

#### 给寺院

✅ **自动数据库**
- Google Sheets 表格，每许一个愿、每还一次愿就自动新增一行
- 可离线查看、导出，无需额外系统

✅ **内容库**
- 一列「同意公开」筛选，瞬间得到 100% 获得同意的故事素材
- 每个故事附带：化名、所求面向、许愿内容、应验过程、感恩之言

✅ **运营杠杆**
- 短视频脚本现成（「一只野象的还愿日记」系列化）
- 社媒文案有情感、有细节、有转发价值
- 比任何自己编造的故事都更有说服力

## 设计亮点

### 1. **四面选择器**（签名设计元素）
- 圆心是金色四面佛祖，四个方向各一面按钮
- 选中时高亮、发光（视觉反馈）
- 映射四面佛「四面朝向四方」的本质，不只是功能，是**仪式感**

### 2. **烛光夜庙的配色**
- 深暖夜色底（#1C1410）
- 金色烛光（#C9A24B，#F0C868）
- 朱漆红（#651A12）
- 陶土米纸（#8A6A4F，#EDE3D0）
- 不是网红粉色，而是**寺院本身的气质**

### 3. **故事卡的庄严性**
- 不像社交 APP 那样吵闹
- 黑底金框，行文如文言，读起来像**一份记录、一份见证**
- 可截图成纪念品，可传播成信息素

### 4. **全流程无 BUG 的交互**
- 不会卡、不会因网络差而丢数据（本地优先）
- 先完成后同步；同步失败数据仍在本机（降级方案）
- 换手机、凭证号、找回，逻辑完整

    desc:'求事业顺遂、学业有成、前程似锦' },
  { id:2, pos:'right',  no:'第二面', seek:'姻缘 · 家庭', 
    desc:'求姻缘和合、家庭美满、贵人相助' },
  { id:3, pos:'bottom', no:'第三面', seek:'财富 · 生意', 
    desc:'求财源广进、生意兴隆、丰衣足食' },
  { id:4, pos:'left',   no:'第四面', seek:'健康 · 平安',

按本寺的说法改 `seek` 和 `desc`。

**说明**：民间常用「事业／姻缘／财富／健康」四类；正统泰传则以「慈悲喜舍四无量心」来释义。请先和法师确认本寺的主张，再定稿。

同文件，约第 235 行：

```javascript
const NICKNAMES = ['一只野象','一盏夜灯','一片菩提叶',...];
```

加进去你想要的化名。建议保留一些诗意的、中性的、不带政治色彩的。

### 改页面文案

页面上的任何文字都可改——搜索找到就改。比如：
- 「还愿日记」改成「愿望记录」
- 「你的化名」改成「你的昵称」

## 技术说明（给开发者）

### 架构

**前端**：纯 vanilla JavaScript + CSS，无框架依赖。
- localStorage 本地存储（浏览器内存、跨页面可用）
- 降级方案：如果 localStorage 不可用（某些预览环境），自动切换到内存存储
- 视图层用 `.active` 类切换（SPA 模式）

**后端**：Google Apps Script（Google 官方免费 serverless）
- HTTP 接口：
  - `POST /exec` —— 接收许愿/还愿数据，appendRow 到表格
  - `GET /exec?id=XXXX&callback=YYYY` —— JSONP 跨域回源，凭证号查询
- no-cors 模式可靠（POST 提交不需等回应，数据已写入；本机也保存一份）
- JSONP 有 8 秒超时，自动降级

**数据格式**

许愿数据（本地结构）：
```javascript
{
  id: "HYP9MY",           // 凭证号
  nickname: "一只野象",
  face: 2,                // 第几面
  faceName: "第二面·姻缘·家庭",
  wish: "愿姻缘和合",
  createdAt: "2026-06-15T10:30:00Z",
  status: "waiting",      // 或 "fulfilled"
  
  // 还愿时填充：
  fulfilledAt: "2026-12-20T14:22:00Z",
  waited: "半年",
  how: "意想不到的邂逅……",
  words: "感恩佛祖庇佑",
  shareConsent: true
}
```

表格一行格式：
| 凭证号 | 化名 | 第几面 | 所求 | 许愿内容 | 许愿时间 | 状态 | 还愿时间 | 等待多久 | 如何应验 | 想说的话 | 同意公开 |

### 安全考虑

✅ **隐私**
- 不收集 IP、不收集地理位置、不跟踪用户
- 所有数据存储在寺院自己的 Google 表格，寺院完全控制
- 前端代码完全开源（可审计）

✅ **防止滥用**
- 后端无认证（但无状态、无敏感操作）
- 凭证号是随机 6 位（2^36 ≈ 688 亿种组合），暴力破解几乎不可能
- 表格行数有上限（Google Sheets 可达 1000 万行），正常寺院数十年不会满

⚠️ **已知限制**
- 如果 Google Apps Script 被 Google 封禁（极罕见），新提交会失败；本机仍可暂存，后续手动导入
- 跨域凭证号查询（换手机找回）依赖网络；离线时本机记录仍可用

### 浏览器兼容性

✅ 在以下环境测试通过：
- iOS Safari 14+
- Android Chrome 80+
- PC Chrome/Firefox/Safari（最近两个版本）

不支持 IE11 及更早（localStorage 等新特性）；不过宗教场景一般用户设备都比较新。

## 常见问题

### Q: 需要后端服务器吗？
**A:** 不需要。用 Google Apps Script 免费部署即可。你只需有一个 Google 账号（个人或寺院账号都可）。

### Q: 信众不联网时怎么办？
**A:** 
- 首次许愿需要网络（提交数据）
- 后续在同一部手机上看「我的专区」、填还愿信息都**不需网络**（本地存储）
- 最后提交还愿信息时再连网
- 换手机用凭证号找回才需网络

### Q: 能否离线部署？
**A:** 
- HTML 可以直接存在寺院的 U 盘，现场打开（无网也能用，但功能受限）
- 建议还是上传到寺院网站（如 lordbuddha.org.sg 的新页面），用二维码扫，更方便

### Q: 可以改成其他寺院的名字吗？
**A:** 当然。搜索 `三宝佛光寺` 改成你的寺院名即可。或搜索 `lordbuddha.org.sg` 改成你的网址。

### Q: 故事被滥用怎么办？
**A:** 
- 每条故事都有「同意公开」的勾选，没勾的不能用
- 寺院有表格访问权，可随时删除数据
- 建议在寺院网站、社交账号声明"所有故事均已获得分享同意，遵守隐私原则"

### Q: 需要服从宗教审查吗？
**A:** 
- 系统本身不设审核机制（信众直接提交）
- 但寺院可以在表格里看到所有记录，有问题的行可以手动删除
- 建议贴一个「分享规则」：只分享信仰与感恩，不宣传任何商业、不承诺保证灵验等

### Q: 可以做公众号版本吗？
**A:** 可以，但需额外开发：
- 公众号 JSSDK 集成（获得用户标识）
- 微信支付接入（可选——如果希望用户打赏、种香火）
- 分享到朋友圈时自动生成漂亮海报

这超出了这套系统的范围，如需要可单独定制。

### Q: 怎样导出故事集？
**A:** 
1. 在表格里筛选「同意公开 = 是」
2. 导出 CSV
3. 用 Python / Excel 脚本生成 Markdown / PDF 的「还愿故事集」
4. 印成册子，或发布成网络文章

（我可以后续帮你写导出脚本。）

## 使用场景 & 效果

### 场景 1：扫码许愿
信众来寺院拜四面佛，义工指引"扫这个二维码，可以把心愿记下来"。信众扫进去，选第几面、写愿望、走人。凭证号截图留着。

### 场景 2：应验后回来还愿
半年后，愿望成真。信众想起来，凭凭证号进来，填"我等了半年，发生了…"，故事卡生成，截图分享给朋友——"看，我的愿望应验了！"

### 场景 3：内容运营
寺院每周筛一遍表格，挑 3 条同意公开的故事：
- Monday：「一只野象的还愿日记」文字版，发微博
- Wednesday：短视频版（配诵经音乐、金色光晕），发 TikTok / IG Reels
- Friday：每月故事集，发 Newsletter / 公众号

三个月下来，粉丝能看到来自真实信众的 36 个还愿故事。这比任何营销文案都能打动人。

### 场景 4：线下汇集
寺院每年一次把年度故事集印成小册子，放在香火区；或做成展板。信众来拜时看到"去年有 127 个心愿应验了"，信心倍增。

## 支持 & 反馈

如遇到问题：

1. **数据没同步到表格？**
   → 检查 `APPS_SCRIPT_URL` 是否粘对了（必须以 `/exec` 结尾）
   → 确认 Google Apps Script 部署时选了"任何人可访问"
   → 检查浏览器控制台有无报错（F12 → Console）

2. **改了 HTML 但看不到效果？**
   → 刷新浏览器缓存（Ctrl+Shift+R）
   → 如果在网站上，清除服务工作者（Settings → Clear Site Data）

3. **想加功能 / 修 Bug？**
   → 代码都是标准 JavaScript + Google Apps Script，注释详细
   → 可以找一个懂前端/Google Script 的开发者协助
   → 或联系我们
